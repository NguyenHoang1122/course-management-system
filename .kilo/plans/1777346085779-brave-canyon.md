# Refactor Plan: Separate Inline CSS/JS from Templates into MVC-Compliant Structure

## Goal
Refactor all remaining HTML templates (excluding home.html and layout.html already handled) to extract inline CSS and JavaScript into separate files, organized in subfolders `static/css/pages/` and `static/js/pages/` for each page. This ensures MVC compliance: Views (HTML) contain only markup, with styles and scripts externalized. Structure mimics real-world projects with page-specific assets.

## Scope
- Templates with inline code: 7 remaining for CSS, 12 for JS (updated from grep).
- Exclude: home.html, layout.html (already refactored), and completed: contact.html, about.html, auth/register.html, course/detail.html, admin/admin-detail.html.
- Target structure:
  - `static/css/pages/` for page-specific CSS (e.g., notifications.css, report-list.css).
  - `static/js/pages/` for page-specific JS (e.g., edit-course.js, notifications.js).
- Update templates: Replace `<style>` with `<link th:href="@{/css/pages/{page}.css}">`, `<script>` with `<script th:src="@{/js/pages/{page}.js}">`.

## Steps
1. **Identify and Extract CSS**:
   - For each remaining template with `<style>`:
     - Read the inline CSS block.
     - Create `static/css/pages/{page}.css` with the content.
     - Remove `<style>` from template.
     - Add `<link rel="stylesheet" th:href="@{/css/pages/{page}.css}">` after core CSS.

2. **Identify and Extract JS**:
   - For each remaining template with `<script>`:
     - Read the inline JS block.
     - Create `static/js/pages/{page}.js` with the content.
     - Remove `<script>` from template.
     - Add `<script th:src="@{/js/pages/{page}.js}"></script>` before `</body>`.

3. **Handle Remaining Templates List**:
   - **CSS Inline (7 files)**: profile/notifications.html, admin/report-list.html, admin/select-lessons.html, profile/wishlist.html, profile/edit.html, course/my-courses.html, admin/admin-edit.html.
   - **JS Inline (12 files)**: admin/edit-course.html, admin/create-course.html, profile/notifications.html, admin/add-lesson.html, admin/lesson-list.html, admin/course-list.html, profile/wishlist.html, auth/login.html, admin/user-trash.html, admin/user-list.html, admin/lesson-detail.html, admin/admin-edit.html.
   - Note: Some templates have both (e.g., profile/notifications.html, profile/wishlist.html, admin/admin-edit.html).

4. **Edge Cases**:
   - If CSS/JS is small, combine into one file per page.
   - Ensure Thymeleaf references work (e.g., `@{/css/pages/{page}.css}`).
   - Preserve layout includes (e.g., `th:replace="~{layout :: headCoreCss}"`).

## Verification
- Run `mvn spring-boot:run` to start app.
- Test pages: Check styles/scripts load (no 404 errors), UI functions (e.g., form validation in register.html).
- Browser console: No JS errors.
- Compare before/after: Templates have no `<style>` or `<script>` blocks.

## Tradeoffs
- Subfolders: Adds organization but increases file count. Alternative: Flat `css/` and `js/` with prefixes (e.g., contact.css). User prefers subfolders for "khoa học" structure – proceed?
- Page-specific: Better cache/separation, but if shared code, consider global files.
